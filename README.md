# Сфера — сайт-візитка

Простий статичний сайт (без збірки, без фреймворків — чистий HTML/CSS)
для промо застосунку «Сфера» і для посилань App Store / Google Play
(Privacy Policy URL тощо).

**Живий сайт:** https://sferaapp.com
**Другий домен:** https://sferaapp.online → 301-редирект на sferaapp.com
(налаштовано на рівні реєстратора, не в цьому репозиторії)
**Репозиторій:** https://github.com/sferaapps-hub/sfera-landing
**Хостинг:** GitHub Pages (безкоштовний, автодеплой з гілки `main`)

## Структура

```
index.html      — головна (лендинг): hero, «як це працює», приватність, CTA
about.html      — «Про застосунок» — детальний опис для App Store/Google Play
privacy.html    — «Правила та конфіденційність» — політика конфіденційності
assets/
  style.css               — єдиний спільний стиль для всіх сторінок
  logo-512.png             — іконка застосунку (в hero/nav)
  apple-touch-icon.png     — 180×180, для iOS
  favicon-32.png           — фавікон
CNAME           — потрібен GitHub Pages, містить "sferaapp.com". НЕ видаляти.
```

Іконка узята з `sfera_app/assets/images/app_icon.png` основного репозиторію
застосунку (той самий, що в самому Flutter-застосунку), змінена у розмірі
через `sips` (macOS), оригінал у git не лежить — тільки готові розміри.

Копія тексту головної сторінки — це той самий макет, що був узгоджений з
власником раніше (Claude-артефакт «Сфера — люди більше не губляться»,
2026-07-16), просто обгорнутий у справжній `<html>` документ і розбитий на
три сторінки з реальною поштою замість плейсхолдера.

## Як редагувати

Це прості HTML-файли — ніякого `npm install`/збірки не потрібно. Правити
можна прямо в текстовому редакторі або просити Claude в новій сесії.

Локальний перегляд:
```bash
cd "~/Сфера - проект"
python3 -m http.server 8934
# відкрити http://localhost:8934/index.html
```

## Як задеплоїти зміни

GitHub Pages деплоїть автоматично при будь-якому push у `main` (~30-60 сек):
```bash
git add -A
git commit -m "опис зміни"
git push
```
Перевірити статус збірки: `gh api repos/sferaapps-hub/sfera-landing/pages/builds/latest`

## Домен і DNS (довідково, змінюється НЕ тут)

Обидва домени куплені на **ukraine.com.ua** (панель ADM.TOOLS), акаунт
власника. DNS для sferaapp.com — панель регістратора → Домени →
sferaapp.com → «Настройки домена (DNS)»:

| Тип | Субдомен | Дані |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | sferaapps-hub.github.io. |

Це стандартні чотири IP GitHub Pages — незмінні, однакові для будь-якого
GitHub Pages сайту. sferaapp.online налаштований через «Веб-редирект» →
«Прямой редирект» → `https://sferaapp.com` (галочка «тільки без www» знята,
щоб www.sferaapp.online теж редиректив).

## GitHub-акаунт

Аккаунт `sferaapps-hub` створено спеціально під цей проєкт 2026-07-18.
`gh` CLI авторизований на машині власника (device flow, без пароля) —
`gh auth status` має показувати залогінений `sferaapps-hub`. Якщо в новій
сесії `gh` не залогінений — треба повторити `gh auth login --web`.
